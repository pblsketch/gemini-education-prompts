# 5. 학생 활용 (Student Use) 프롬프트

학생들이 자기주도 학습, 시험 준비, 글쓰기 개선, 코딩 학습에 활용할 수 있는 프롬프트 가이드입니다.

## 핵심 원칙

### 학생 활용 프롬프트의 특성

| 특성 | 설명 |
|------|------|
| **학습자 중심** | 학생이 스스로 사용할 수 있는 명확한 지시 |
| **안내적 학습** | 답을 직접 주지 않고 이해를 돕는 방향 |
| **메타인지 촉진** | 자신의 학습 과정을 성찰하도록 유도 |
| **안전성** | 학생 연령에 적합한 콘텐츠 보장 |

### PARTS 적용 (학생 관점)

| 요소 | 학생 활용 맥락에서의 적용 |
|------|--------------------------|
| **P: Persona** | 선생님, 튜터, 학습 코치, 스터디 파트너 |
| **A: Act** | 설명해줘, 도와줘, 가르쳐줘, 확인해줘 |
| **R: Recipient** | "나는 ~학년 학생이야" |
| **T: Theme** | 학습 중인 구체적 개념이나 과제 |
| **S: Structure** | 단계별 설명, 예시 포함, 퀴즈 형식 |

## 프롬프트 유형별 가이드

### 5.1 개념 이해 안내 (Understanding Concepts)

#### 기본 템플릿 (학생용)

```
당신은 [교과] 선생님이에요. 저는 [학년] 학생이고, [개념/주제]를 이해해야 해요.

[구체적인 어려움이나 질문]

다음과 같이 설명해주세요:
- 쉬운 말로 설명
- 예시를 들어서
- 단계별로 차근차근
```

#### 예시 프롬프트

📌 **과학 개념 이해**

```
당신은 과학 선생님이에요. 저는 중학교 2학년 학생이고, DNA 복제 과정을 이해해야 해요.

교과서를 읽었는데 헬리케이스, DNA 중합효소 같은 용어가 헷갈려요.

다음과 같이 설명해주세요:
- 각 단계에서 무슨 일이 일어나는지 쉬운 말로
- 지퍼나 복사기 같은 비유를 사용해서
- 핵심 용어의 역할을 하나씩

설명 후에 제가 이해했는지 확인하는 질문 2개도 해주세요.
```

✨ **교사용 활용 팁:**
- 이 프롬프트를 학생들에게 템플릿으로 제공하세요
- "Guided Learning 모드"를 사용하면 더 효과적입니다

🔥 **적합한 상황:**
- 수업 후 복습할 때
- 숙제하다 막힐 때
- 시험 준비할 때

### 5.2 플래시카드 생성 (Flashcard Creation)

#### 예시 프롬프트

📌 **시험 대비 플래시카드**

```
교육 콘텐츠 제작자처럼 행동해서 [교과] 학생을 위한 플래시카드를 만들어줘.

**내가 공부하는 내용:** [수업 자료 첨부 또는 주제 설명]
**학년:** [학년]
**형식:** 질문-답변 플래시카드

다음을 포함해줘:
- 핵심 용어 정의 카드 (10개)
- 개념 이해 확인 카드 (5개)
- 적용 문제 카드 (5개)

각 카드는 이렇게 만들어줘:
[앞면] 질문 또는 용어
[뒷면] 답변 또는 정의

암기하기 쉽게 키워드를 강조해줘.
```

### 5.3 시험 준비 퀴즈 (Exam Preparation)

#### 예시 프롬프트

📌 **맞춤형 연습 퀴즈**

```
당신은 [교과] 선생님이에요. 저는 [학년] 학생이고 [시험 종류] 시험이 다가왔어요.

다음 개념들에 대한 연습 퀴즈를 만들어주세요:
- [어려운 개념 1]
- [어려운 개념 2]
- [어려운 개념 3]

퀴즈 형식:
- 객관식 5문항
- 단답형 3문항
- 서술형 2문항

각 문항의 난이도를 표시해주시고, 정답과 해설도 함께 주세요.
단, 해설은 처음에는 숨겨두고 제가 먼저 풀어본 후에 보여주세요.
```

### 5.4 글쓰기 피드백 (Writing Feedback)

#### 예시 프롬프트

📌 **에세이 피드백 요청**

```
당신은 글쓰기 선생님이에요. 제가 쓴 글을 봐주시고 피드백을 주세요.

**과제:** [과제 설명]
**글의 종류:** [논술/감상문/보고서/자기소개서]
**제 글:** [글 첨부 또는 입력]

다음에 대해 피드백해주세요:
1. 주장이 명확한지
2. 근거가 충분한지
3. 문단 구성이 자연스러운지
4. 문장 표현에서 개선할 점

중요: 고쳐진 글을 바로 주지 말고, 제가 스스로 고칠 수 있게 가이드만 해주세요.
어떤 부분을 어떻게 고치면 좋을지 방향을 알려주세요.
```

✨ **교사용 활용 팁:**
- "수정본을 주지 말고 가이드만"이라는 지시가 중요합니다
- 학생이 스스로 개선하도록 유도하는 피드백 문화를 형성하세요

### 5.5 프로젝트 아이디어 브레인스토밍

#### 예시 프롬프트

📌 **연구 주제 탐색**

```
당신은 연구 도서관 사서처럼 행동해주세요. 저는 [학년] 학생이고 [교과] 수업에서 10페이지 연구 보고서를 써야 해요.

**관심 분야:** [관심 있는 주제나 영역]
**제한 조건:** [주어진 조건이 있다면]

다음을 도와주세요:
1. 연구 가능한 구체적인 주제 3가지 제안
2. 각 주제별로 왜 좋은 연구 주제인지 설명
3. 각 주제에서 탐구할 수 있는 연구 질문 2개씩
4. 시작할 때 찾아볼 자료 유형 제안

제가 관심 있는 주제를 선택하면, 더 구체적으로 발전시키는 것을 도와주세요.
```

### 5.6 코딩 학습 (Coding Practice)

#### 예시 프롬프트

📌 **프로그래밍 개념 학습**

```
당신은 프로그래밍 선생님이에요. 저는 고등학생이고 Python 기초를 배우고 있어요.

[배우려는 개념: 예) for 반복문]을 이해하고 싶어요.

다음과 같이 가르쳐주세요:
1. 이 개념이 뭔지 쉬운 말로 설명
2. 실생활 비유로 설명
3. 가장 간단한 예제 코드
4. 코드의 각 줄이 무슨 의미인지 설명
5. 제가 직접 수정해볼 수 있는 연습 문제

코드를 직접 작성하지 말고, Canvas를 사용해서 제가 실시간으로 수정하면서 배울 수 있게 해주세요.
```

### 5.7 숙제 도움 (Homework Help)

#### 예시 프롬프트

📌 **수학 문제 풀이 안내**

```
당신은 수학 튜터예요. 저는 중학교 [학년] 학생이고 숙제를 하다가 막혔어요.

**문제:** [문제 입력 또는 사진 첨부]
**내가 시도한 것:** [지금까지 푼 과정 설명]
**막힌 부분:** [어디서 막혔는지]

중요: 답을 바로 알려주지 말고, 다음 단계로 갈 수 있게 힌트만 주세요.
제가 스스로 풀 수 있도록 안내하는 질문을 해주세요.

만약 제가 맞게 풀면 칭찬해주시고, 틀리면 어디서 틀렸는지 알려주세요.
```

✨ **교사용 활용 팁:**
- "Guided Learning 모드"를 사용하면 소크라테스식 질문으로 안내합니다
- 학생들에게 "답을 바로 달라고 하면 안 된다"는 프롬프트 규칙을 가르치세요

### 5.8 자기주도 학습 계획

#### 예시 프롬프트

📌 **시험 준비 계획**

```
당신은 학습 코치예요. 저는 [학년] 학생이고 2주 후에 [과목] 중간고사가 있어요.

**시험 범위:** [범위 설명]
**어려운 부분:** [특히 어려운 단원이나 개념]
**공부 가능 시간:** 평일 2시간, 주말 4시간

2주 공부 계획을 세워주세요:
- 일자별 학습 내용
- 어려운 부분에 더 많은 시간 배분
- 복습 주기 포함
- 모의고사 또는 자가 테스트 일정
- 체크리스트 형식으로

또한, 공부하다 지칠 때 쉬는 방법도 알려주세요.
```

## 학생용 프롬프트 작성 가이드

### 좋은 학생 프롬프트의 특징

| 요소 | 좋은 예 | 피해야 할 예 |
|------|--------|-------------|
| **역할 지정** | "당신은 과학 선생님이에요" | (역할 없음) |
| **자기 소개** | "저는 중2 학생이에요" | "DNA 설명해줘" |
| **구체적 요청** | "헬리케이스 역할을 쉽게 설명해줘" | "DNA 복제 알려줘" |
| **학습 방식** | "예시를 들어서 단계별로" | (방식 없음) |
| **정직한 어려움** | "이 부분이 헷갈려요" | "다 모르겠어요" |

### 학생이 피해야 할 프롬프트

❌ **답만 요청**: "이 문제 답이 뭐야?"
❌ **과제 대신 해달라기**: "이 에세이 써줘"
❌ **너무 모호함**: "수학 도와줘"
❌ **표절 의도**: "이 글을 내가 쓴 것처럼 바꿔줘"

### 학생이 사용해야 할 프롬프트

✅ **이해 요청**: "이 개념을 이해하게 도와줘"
✅ **힌트 요청**: "이 문제 푸는 방향을 알려줘"
✅ **피드백 요청**: "내 글에서 개선할 점을 알려줘"
✅ **확인 요청**: "내가 이해한 게 맞는지 확인해줘"

## 교사가 학생에게 제공할 프롬프트 템플릿

### 수업 중 활용 템플릿

```markdown
## 📚 [교과] 학습 도우미 프롬프트

### 개념 이해가 필요할 때
"당신은 [교과] 선생님이에요. 저는 [학년] 학생이고, [개념]을 이해해야 해요. 쉬운 말과 예시로 설명해주세요. 설명 후에 이해 확인 질문도 해주세요."

### 문제 풀이 도움이 필요할 때
"당신은 [교과] 튜터예요. 이 문제를 풀다가 막혔어요. [막힌 부분] 답을 바로 주지 말고, 다음 단계로 갈 힌트를 주세요."

### 내 답을 확인하고 싶을 때
"당신은 [교과] 선생님이에요. 제가 푼 답이 맞는지 확인해주세요. 틀렸다면 어디서 틀렸는지 알려주시고, 맞았다면 왜 맞는지 설명해주세요."
```

## 교사용 지도 사항

### AI 리터러시 교육 포인트

1. **AI의 한계 인식**: AI도 틀릴 수 있음
2. **비판적 검토**: AI 답변을 맹신하지 않기
3. **학습 목적**: 답을 얻는 것이 아니라 이해하는 것이 목적
4. **정직한 사용**: AI 도움받은 부분 명시하기
5. **개인정보 보호**: 개인 정보 입력하지 않기

### 학급 규칙 예시

```
📋 AI 학습 도구 사용 규칙

1. AI는 "선생님의 보조"입니다. 최종 확인은 선생님께!
2. "답 알려줘" 대신 "이해하게 도와줘"로 질문합니다.
3. AI 도움받은 과제는 반드시 표시합니다.
4. 이름, 학교, 연락처 등 개인정보는 입력하지 않습니다.
5. AI 답변도 틀릴 수 있으니 교과서로 확인합니다.
```

## 프롬프트 품질 체크리스트

학생용 프롬프트 제공 전 확인:

- [ ] 학생이 직접 사용할 수 있는 쉬운 표현인가?
- [ ] 답을 직접 주지 않고 학습을 안내하는가?
- [ ] 학년 수준에 적합한 언어인가?
- [ ] 메타인지를 촉진하는 요소가 있는가?
- [ ] 안전하고 적절한 콘텐츠 생성을 유도하는가?
- [ ] 학생의 정직한 학습을 장려하는가?

## Gemini 고급 기능 학생 활용

### Guided Learning 모드
- 답 대신 단계별 이해 구축
- 소크라테스식 질문으로 안내
- 시험 준비, 숙제 도움에 최적

### Canvas 모드
- 코딩 프로젝트 실시간 실행
- 글쓰기 실시간 편집
- 학습 자료 협업 제작

### Deep Research 모드
- 연구 보고서 자료 수집
- 프로젝트 주제 탐구
- 문헌 검토 지원
