# NotebookLM 연계 활용 가이드

Gemini 프롬프트와 NotebookLM의 연계 활용 방법을 안내합니다.

## NotebookLM 개요

### 핵심 특징

| 특징 | 설명 |
|------|------|
| **소스 기반 응답** | 업로드한 자료에만 기반하여 답변 |
| **인라인 인용** | 모든 답변에 출처 표시 |
| **다국어 지원** | 50개 이상 언어 지원 |
| **다양한 출력** | 요약, 스터디 가이드, 퀴즈, FAQ, 오디오 오버뷰 |

### Gemini vs NotebookLM

| 측면 | Gemini | NotebookLM |
|------|--------|------------|
| **정보 소스** | 웹 + 학습 데이터 | 업로드한 자료만 |
| **정확성** | 일반적 | 소스 기반으로 높음 |
| **인용** | 웹 검색 시 가능 | 항상 인용 포함 |
| **창의성** | 높음 | 소스 범위 내 |
| **적합한 용도** | 아이디어 생성, 일반 질문 | 특정 자료 분석, 정확한 정보 필요 시 |

## 교육 활용 워크플로우

### 워크플로우 1: 수업 준비

```
1. NotebookLM에 소스 업로드
   - 교과서 관련 부분
   - 교육과정 문서
   - 보충 자료
   ↓
2. NotebookLM에서 내용 파악
   - "이 단원의 핵심 개념 5가지는?"
   - "학생들이 어려워할 개념은?"
   ↓
3. Studio 패널에서 자동 생성
   - 스터디 가이드
   - FAQ
   - 마인드맵
   ↓
4. Gemini에서 수업 설계 프롬프트
   - NotebookLM 분석 결과를 바탕으로 구체적 프롬프트 작성
```

### 워크플로우 2: 학생 학습 지원

```
1. 교사: NotebookLM 노트북 생성
   - 단원 핵심 자료 업로드
   - 스터디 가이드, 퀴즈 미리 생성
   ↓
2. 교사: 노트북 공유 또는 Classroom 연결
   ↓
3. 학생: NotebookLM에서 질문
   - 개념 질문
   - 예시 요청
   - 비교 분석
   ↓
4. 학생: 자동 생성 도구 활용
   - 오디오 오버뷰로 복습
   - 스터디 가이드로 정리
```

### 워크플로우 3: 평가 개발

```
1. NotebookLM에 평가 관련 소스 업로드
   - 성취기준 문서
   - 교과서/학습 자료
   - 기존 평가 예시
   ↓
2. NotebookLM에서 내용 분석
   - "이 성취기준의 핵심 평가 요소는?"
   - "어떤 유형의 문항이 적합할까?"
   ↓
3. Gemini에서 평가 프롬프트
   - NotebookLM 분석 결과 기반으로 구체적 문항 요청
```

## NotebookLM 프롬프트 예시

### 수업 설계 지원

📌 **단원 핵심 개념 추출**

```
업로드한 교육과정 문서와 교과서 내용을 바탕으로:
1. 이 단원의 핵심 개념 5가지를 정리해주세요
2. 각 개념 간의 연결 관계를 설명해주세요
3. 학생들이 가장 어려워할 개념은 무엇일까요?
4. 선수 학습으로 필요한 개념은 무엇인가요?
```

📌 **수업 시퀀스 제안**

```
업로드한 자료를 기반으로, 이 단원을 5차시로 가르친다면 
어떤 순서로 개념을 도입하는 것이 좋을까요?

각 차시별로:
- 핵심 학습 내용
- 연결되는 이전 차시 개념
- 다음 차시를 위한 준비
```

### 평가 개발 지원

📌 **평가 문항 아이디어**

```
업로드한 성취기준과 학습 자료를 바탕으로:
1. 이 단원에서 평가해야 할 핵심 역량은 무엇인가요?
2. 형성평가로 적합한 질문 유형 3가지를 제안해주세요
3. 수행평가 과제로 적합한 활동을 제안해주세요
```

📌 **오개념 파악**

```
이 단원에서 학생들이 흔히 보이는 오개념은 무엇일까요?
각 오개념에 대해:
- 오개념의 내용
- 왜 이런 오개념이 생기는지
- 이를 확인할 수 있는 진단 문항 예시
```

### 차별화 지원

📌 **수준별 설명**

```
업로드한 자료의 [특정 개념]을 세 가지 수준으로 설명해주세요:
1. 기초 수준: 핵심만 간단히
2. 표준 수준: 교과서 수준
3. 심화 수준: 확장 개념 포함
```

📌 **핵심 어휘 정리**

```
이 단원의 핵심 학습 어휘 목록을 만들어주세요:
- 어휘
- 정의 (학생 친화적)
- 예시 또는 비유
- 관련 어휘
```

### 학생 활용

📌 **개념 질문**

```
[특정 개념]이 잘 이해가 안 돼요.
1. 쉬운 말로 설명해주세요
2. 실생활 예시를 들어주세요
3. 이 개념이 왜 중요한지 알려주세요
```

📌 **시험 준비**

```
이 단원 시험을 준비하려고 해요.
1. 가장 중요한 개념 5가지는 뭔가요?
2. 자주 출제되는 유형은 어떤 건가요?
3. 연습 문제를 만들어줄 수 있나요?
```

## Studio 패널 활용

### 자동 생성 기능

| 기능 | 생성물 | 활용 |
|------|--------|------|
| **Study guide** | 스터디 가이드 | 학생 복습용 |
| **FAQ** | 자주 묻는 질문 | 예상 질문 파악 |
| **Timeline** | 타임라인 | 역사, 순서 정리 |
| **Mind Map** | 마인드맵 | 개념 관계 시각화 |
| **Audio Overview** | 오디오 요약 | 이동 중 학습 |
| **Video Overview** | 영상 요약 | 시각적 학습 |

### 사용 방법

```
1. 소스 선택 (전체 또는 특정 소스)
   ↓
2. Studio 패널에서 원하는 출력 선택
   ↓
3. "Generate" 클릭
   ↓
4. 결과 검토 및 저장
   ↓
5. 필요시 "Customize" 옵션으로 조정
```

### Customize 활용

오디오/비디오 오버뷰 생성 시 맞춤 설정:

```
"학생 친화적인 어조로 만들어주세요"
"핵심 용어 정의에 집중해주세요"
"실생활 예시를 많이 포함해주세요"
"[특정 주제]에 초점을 맞춰주세요"
```

## Gemini + NotebookLM 연계 프롬프트

### 패턴 1: NotebookLM 분석 → Gemini 생성

```
[NotebookLM에서 얻은 분석 결과를 Gemini 프롬프트에 포함]

예시:
"NotebookLM에서 분석한 결과, 이 단원의 핵심 개념은 
1) DNA 구조, 2) 복제 과정, 3) 유전 정보 전달입니다.

이 분석을 바탕으로 5E 모형 수업 계획을 만들어주세요.
각 개념이 어느 단계에서 다뤄지는지 명시해주세요."
```

### 패턴 2: Gemini 생성 → NotebookLM 검증

```
[Gemini가 생성한 내용을 NotebookLM으로 검증]

1. Gemini에서 수업 계획 또는 평가 문항 생성
2. NotebookLM에 교육과정 문서 업로드
3. NotebookLM에서 질문:
   "이 수업 계획/평가 문항이 [성취기준]에 적절히 정렬되어 있나요?"
```

### 패턴 3: 상호 보완 활용

| 단계 | 도구 | 활동 |
|------|------|------|
| 1 | NotebookLM | 교과서/교육과정 분석 |
| 2 | Gemini | 수업 아이디어 브레인스토밍 |
| 3 | NotebookLM | 아이디어의 교육과정 정합성 확인 |
| 4 | Gemini Canvas | 수업 계획 상세 작성 |
| 5 | NotebookLM | 학생용 스터디 가이드 생성 |

## 교사용 NotebookLM 노트북 설계

### 추천 노트북 구조

#### 단원별 노트북

```
📁 [교과] [단원명] 노트북
├── 📄 교육과정 성취기준
├── 📄 교과서 해당 부분
├── 📄 보충 자료 1
├── 📄 보충 자료 2
└── 📄 기존 수업 계획 (참고용)
```

#### 평가 개발 노트북

```
📁 [교과] 평가 자료 노트북
├── 📄 성취기준 문서
├── 📄 평가 기준 가이드
├── 📄 기존 평가 문항 예시
└── 📄 루브릭 예시
```

### 소스 업로드 팁

- **적절한 양**: 노트북당 10-20개 소스 권장
- **품질 우선**: 핵심 자료만 선별
- **형식 다양화**: PDF, Google Docs, 웹 링크 혼합
- **명확한 명명**: 소스 이름 명확하게 지정

## Classroom 연계

### NotebookLM을 Classroom에 연결

```
1. Classroom에서 "+" → "Material" 선택
2. "Attach" → NotebookLM 아이콘 선택
3. 노트북 선택 (기존 또는 새로 생성)
4. "Highlight at top of class" 체크 (선택)
5. "Post" 클릭
```

### 학생 접근 관리

- 학생은 "View only" 모드로 접근
- 소스 추가/삭제 불가
- 교사가 생성한 스터디 가이드, 오디오 오버뷰 활용 가능
- 교사는 "Opened by" 상태로 접근 현황 확인

## 문제 해결

### 자주 발생하는 문제

| 문제 | 원인 | 해결 |
|------|------|------|
| 관련 없는 답변 | 소스에 정보 부족 | 관련 소스 추가 |
| 인용 누락 | 질문이 너무 일반적 | 더 구체적으로 질문 |
| 생성 오류 | 소스 형식 문제 | 다른 형식으로 변환 |
| 느린 응답 | 소스 과다 | 불필요한 소스 제거 |

### 최적화 팁

1. **소스 품질**: 명확하고 구조화된 문서 사용
2. **적절한 범위**: 질문과 관련된 소스만 선택
3. **구체적 질문**: 모호한 질문 피하기
4. **검증 습관**: 중요한 정보는 원본 소스에서 확인
