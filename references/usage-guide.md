# 깊이 있는 수업과 평가를 위한 Gemini 프롬프트 활용 가이드

스킬의 효과적인 활용을 위한 종합 가이드입니다.

## 스킬 사용 워크플로우

### 기본 워크플로우

```
1. 사용자 요청 분석
   ↓
2. 6가지 영역 중 해당 영역 파악
   - 수업 설계 / 평가 / 수업 자료 / 차별화 / 학생 활용 / 심층 연구
   ↓
3. 필수 정보 확인
   - 학교급, 학년, 교과
   - 성취기준 또는 학습 목표
   - 학급/학생 특성
   - 원하는 출력 형식
   ↓
4. references 파일 참조
   - 해당 영역의 상세 가이드 확인
   - 템플릿과 예시 활용
   ↓
5. PARTS 프레임워크 적용
   - 5요소 모두 포함 확인
   ↓
6. 한국 교육 맥락화
   - 성취기준 연계
   - 수업 시간, 학급 규모 반영
   ↓
7. 프롬프트 제공
   - 복사-붙여넣기 가능한 형태
   - 활용 팁 포함
   - 후속 프롬프트 제안
```

### 사용자 유형별 가이드

#### 교사 사용자

| 요청 유형 | 우선 확인 사항 | 제공 형식 |
|----------|--------------|----------|
| 수업 계획 | 학년, 성취기준, 차시 | 단계별 수업 구조 |
| 평가 개발 | 평가 목적, 문항 유형 | 문항 + 정답/루브릭 |
| 자료 제작 | 대상 수준, 분량 | 한글 호환 형식 |
| 차별화 | 학생 특성, 지원 영역 | 수준별 활동 세트 |

#### 학생 사용자

| 요청 유형 | 주의 사항 | 제공 형식 |
|----------|----------|----------|
| 개념 이해 | 답 직접 제공 X | 안내적 설명 |
| 숙제 도움 | 대신 해주기 X | 힌트와 가이드 |
| 시험 준비 | 학습 촉진 중심 | 연습 문제 + 자기점검 |
| 글쓰기 | 대필 X | 피드백 + 개선 방향 |

#### 교육 연구자 사용자

| 요청 유형 | 추천 기능 | 제공 형식 |
|----------|----------|----------|
| 문헌 검토 | Deep Research | 인용 포함 보고서 |
| 방법론 탐구 | Deep Research | 비교 분석표 |
| 정책 분석 | Deep Research | 사례 비교 |

## 영역별 퀵 가이드

### 1. 수업 설계 (Planning)

**트리거 키워드:** 수업 계획, 단원 설계, 차시 계획, 교육과정

**필수 정보:**
- 학교급/학년
- 교과/단원명
- 성취기준
- 수업 모형 (선택)
- 차시/기간

**핵심 페르소나:** 교육과정 설계 전문가, 교수설계자

**참조:** [category1-planning.md](category1-planning.md)

---

### 2. 평가 개발 (Assessment)

**트리거 키워드:** 퀴즈, 시험, 평가, 루브릭, 문항, 수행평가

**필수 정보:**
- 평가 목적 (형성/총괄/진단)
- 평가 범위/성취기준
- 문항 유형/수
- 채점 기준 필요 여부

**핵심 페르소나:** 평가 설계 전문가, 문항 개발자

**참조:** [category2-assessment.md](category2-assessment.md)

---

### 3. 수업 자료 (Instructional Materials)

**트리거 키워드:** 학습지, 활동지, 실험 가이드, 토론 자료, 읽기 자료

**필수 정보:**
- 자료 유형
- 대상 학년/수준
- 분량
- 활용 맥락

**핵심 페르소나:** 학습 자료 개발자, 콘텐츠 편집자

**참조:** [category3-instructional-materials.md](category3-instructional-materials.md)

---

### 4. 차별화 (Differentiation)

**트리거 키워드:** 수준별, 개별화, 피드백, 지원, UDL, 다문화

**필수 정보:**
- 학습자 특성/수준
- 지원 영역
- 공통 학습 목표
- 차별화 방식

**핵심 페르소나:** 학습 코치, 특수교육 전문가

**참조:** [category4-differentiation.md](category4-differentiation.md)

---

### 5. 학생 활용 (Student Use)

**트리거 키워드:** 학생용 프롬프트, 자기주도 학습, 숙제 도움, 시험 준비

**필수 정보:**
- 학생 학년
- 학습 상황
- 필요한 지원 유형

**핵심 페르소나:** 튜터, 학습 코치

**참조:** [category5-student-use.md](category5-student-use.md)

---

### 6. 심층 연구 (Deep Research)

**트리거 키워드:** 문헌 검토, 연구 동향, 비교 분석, 정책 분석

**필수 정보:**
- 연구 질문
- 검토 범위
- 분석 관점
- 보고서 형식

**핵심 페르소나:** 연구 보조원, 정책 분석가

**참조:** [category6-deep-research.md](category6-deep-research.md)

## 정보 수집 가이드

### 필수 정보가 부족할 때

사용자에게 다음을 확인하세요:

```
프롬프트를 만들기 위해 몇 가지 정보가 필요합니다:

1. **학교급/학년**: 어느 학년을 대상으로 하나요?
2. **교과/주제**: 어떤 교과의 어떤 내용인가요?
3. **성취기준**: 관련 성취기준이 있나요? (선택)
4. **학급 특성**: 특별히 고려해야 할 학생 특성이 있나요?
5. **원하는 형식**: 특정 형식이나 모형이 있나요?
```

### 정보 수집 질문 예시

| 영역 | 핵심 질문 |
|------|----------|
| 수업 설계 | "몇 차시로 계획하시나요?" "선호하는 수업 모형이 있나요?" |
| 평가 | "형성평가인가요, 총괄평가인가요?" "어떤 유형의 문항을 원하시나요?" |
| 자료 | "학습지 분량은 어느 정도가 적당할까요?" "수업 중 사용인가요, 과제용인가요?" |
| 차별화 | "어떤 특성의 학생들을 지원하려고 하시나요?" |
| 학생용 | "학생이 직접 사용할 프롬프트인가요, 교사가 제공할 것인가요?" |

## 품질 관리 체크리스트

### 프롬프트 제공 전 최종 점검

#### PARTS 완성도
- [ ] P: 역할이 구체적이고 적합한가?
- [ ] A: 행동 동사가 명확한가?
- [ ] R: 대상이 구체적으로 명시되었는가?
- [ ] T: 주제가 성취기준과 연결되었는가?
- [ ] S: 출력 형식이 요청되었는가?

#### 한국 교육 맥락
- [ ] 2022 개정 교육과정 성취기준 형식 사용
- [ ] 한국 학교 시간(45분/50분) 반영
- [ ] 한국 학급 규모(25-30명) 고려
- [ ] 한국어 존댓말 체계 반영

#### 실용성
- [ ] 복사-붙여넣기로 바로 사용 가능
- [ ] 한글 파일 호환 형식
- [ ] 불필요한 영어 용어 최소화
- [ ] 활용 팁 포함

#### 확장성
- [ ] 후속 프롬프트 제안 포함
- [ ] 수정/조정 방향 안내
- [ ] 관련 영역 프롬프트 연계

## 자주 묻는 질문 (FAQ)

### Q1: Gemini에서 어떤 모드를 사용해야 하나요?

| 상황 | 권장 모드 |
|------|----------|
| 일반 수업 계획 | 기본 모드 |
| 코딩, 문서 작업 | Canvas 모드 |
| 학생 학습 지원 | Guided Learning 모드 |
| 심층 연구, 문헌 검토 | Deep Research 모드 |
| 반복 작업, 특정 역할 | Gems (맞춤 AI) |

### Q2: 프롬프트가 너무 길어지면 어떻게 하나요?

- 핵심 요소만 포함하고 세부 사항은 후속 프롬프트로
- 단계별로 나누어 요청 (예: 먼저 구조, 그 다음 내용)
- Gems로 자주 사용하는 설정을 저장

### Q3: 결과가 마음에 들지 않으면?

1. **구체성 추가**: "더 구체적으로", "예시를 들어서"
2. **형식 변경**: "표 형식으로", "단계별로"
3. **수준 조정**: "더 쉽게", "더 전문적으로"
4. **재생성**: "다른 접근으로 다시"

### Q4: 한국 성취기준 형식은 어떻게 쓰나요?

```
형식: [학년군/학교급 코드 + 교과 코드 + 영역 번호 + 성취기준 번호]

예시:
- [9국02-03]: 중학교 국어 읽기 영역 3번
- [10영01-04]: 고등학교 영어 듣기 영역 4번
- [6과08-02]: 초등 5-6학년 과학 지구와 우주 2번
```

## 관련 Gemini 기능 연계

### NotebookLM 연계

```
활용 시나리오:
1. 교과서, 교육과정 문서를 NotebookLM에 업로드
2. NotebookLM에서 내용 기반 질문/요약
3. 결과를 바탕으로 Gemini에서 프롬프트로 수업 설계
```

### Gems 활용

```
추천 Gem 유형:
- 교과별 수업 설계 Gem
- 평가 문항 개발 Gem
- 학생 피드백 코치 Gem
- 차별화 전략 조언 Gem
```

### Canvas 활용

```
적합한 상황:
- 수업 계획 실시간 편집
- 학습지 협업 제작
- 코딩 교육 자료 개발
```
